
# GWC Root Sim — 2D r-z 기반 에어프루닝 팟 최적화 엔진 설계서 v2

> Garden with Couch 에어프루닝 팟 시뮬레이터
> 
> **이 문서의 철학**: 처음부터 완벽을 추구하지 않는다. 기본적이고 필수인 것만 확실하게 구현하고, 나중에 필요한 것은 언제든 끼워넣을 수 있는 구조로 만든다. 검증할 파라미터를 최소화해서 "이게 왜 이렇게 나왔지?"를 항상 명확히 추적할 수 있게 한다.
> 
> **MVP 한 줄 목표**: 2D r-z 단면에서, 에어룸 배치가 뿌리 프루닝 위치와 뿌리 표면적에 어떤 영향을 주는지 빠르게 비교하는 1차 설계 탐색기를 만든다.

---

## 0. 이 문서를 읽는 법

이 문서는 두 가지 색으로 구분된다.

- **[MVP]** 표시: 지금 당장 구현할 것. 이것만 만들면 시뮬레이터가 작동한다.
- **[확장]** 표시: 나중에 추가할 것. 지금은 자리(인터페이스)만 만들어두고 비워둔다.

구현 순서는 11장 마일스톤을 따른다. MVP를 먼저 완성하고, 작동을 확인한 뒤에만 확장으로 넘어간다.

## 1. 전체 아키텍처

### 1.1 데이터 흐름

```
입력 (config.json)
   ↓
[MVP] Phase A — 공간 복셀화 (2D r-z 격자 생성)
   ↓
[MVP] Phase B — 뿌리 성장 + 에어프루닝
   ↓
[MVP] Phase C — 점수 계산
   ↓
[MVP] Phase D — 랜덤 탐색 (100개 비교 → Top 5)
   ↓
[MVP] Phase E — 시각화
   ↓
[확장] Phase F — 자원 확산 (Cellular Automata)
[확장] Phase G — 자원 흡수 (Mass Balance)
[확장] Phase H — 유전 알고리즘 (랜덤 탐색을 진화로 업그레이드)
```

### 1.2 핵심 설계 원칙

1. **MVP는 자원 확산 없이 시작한다.** 우리 제품의 핵심 IP는 "물이 어떻게 흐르냐"가 아니라 "에어룸 구조가 프루닝을 어떻게 유도하느냐"다. 따라서 에어룸 접촉 → 프루닝 → 표면적 증가 → 점수화가 가장 먼저 작동해야 한다. 자원 확산은 그 다음이다.
    
2. **MVP는 유전 알고리즘 없이 시작한다.** 먼저 랜덤하게 100개 설계를 만들어 점수를 비교한다. 이게 잘 작동하면(= 점수 함수가 신뢰할 만하면) 그때 유전 알고리즘으로 업그레이드한다. 랜덤 탐색과 유전 알고리즘은 코드 구조가 거의 같아서 나중에 바꾸기 쉽다.
    
3. **흡수량(ml/mg) 계산은 MVP에서 뺀다.** 흡수량을 계산하려면 식물별 K값, Km 같은 파라미터가 필요한데, 우리는 아직 정확한 값이 없다(추정값뿐). 검증할 파라미터가 늘어나면 결과를 신뢰하기 어렵다. MVP는 "뿌리 표면적 + 프루닝 횟수"로만 비교한다. 이 둘은 에어룸 배치만으로 결정되므로 검증이 단순하다.
    

### 1.3 모듈 구조

|모듈|파일|단계|책임|
|---|---|---|---|
|Config|`config.py`|[MVP]|입력 파라미터 로드/검증, 기본값 관리|

## 2. [MVP] Phase A — 공간 복셀화

### 2.1 좌표계

화분이 중심축 기준으로 회전 대칭이라고 가정한다(에어룸이 방사형으로 균등 배치되면 어느 각도로 잘라도 단면이 비슷하다). 따라서 3D 전체가 아니라 2D 단면만 시뮬한다.

- `r` (가로축): 화분 중심축에서의 거리. 범위 0 ~ `pot_radius`
- `z` (세로축): 바닥에서의 높이. 범위 0 ~ `pot_height`
- 한 복셀의 크기: `voxel_size` (기본 0.5cm)

예: 반지름 12cm, 높이 20cm, 복셀 0.5cm면 격자는 24 × 40 = 960칸.

**왜 2D인가**: 3D로 0.5mm 격자를 만들면 한 화분당 수천만 칸이 되어 일반 PC로는 불가능하다. 2D 단면은 회전 대칭 가정 덕분에 연산량이 약 1/360로 줄면서도 핵심 정보(에어룸 위치, 뿌리 경로, 프루닝 깊이)를 다 담는다.

### 2.2 복셀 속성 (MVP는 최소한만)

각 복셀이 가지는 정보:

```
[MVP] soil_type   : 흙 종류 ID (0=배양토, 1=펄라이트)
[MVP] is_airroom  : 에어룸이면 True (물/뿌리가 들어갈 수 없는 공간)
[MVP] root_visits : 이 복셀을 지나간 뿌리 개수 (성장 후 채워짐)

[확장] theta      : 수분 함량 (자원 확산 단계에서 추가)
[확장] oxygen     : 산소 지수 (자원 확산 단계에서 추가)
[확장] solute     : 양분 농도 (자원 흡수 단계에서 추가)
```

MVP에서 `theta`, `oxygen`, `solute`를 빼는 이유: 이들은 자원 확산/흡수 단계에서만 쓰인다. MVP는 "에어룸 위치 vs 뿌리 경로"만 보므로 흙 종류와 에어룸 여부만 있으면 충분하다. 속성을 적게 두면 디버깅할 때 추적할 변수가 줄어든다.

### 2.3 흙 배합 (MVP는 2종만)

```
[MVP] 흙 종류 2가지:
  - 배양토 (soil_type=0): 보수력 높음, 통기성 낮음
  - 펄라이트 (soil_type=1): 보수력 낮음, 통기성 높음

[MVP] 배합 방식: 단순 비율
  예: {"배양토": 0.6, "펄라이트": 0.4}
  → 각 복셀에 확률적으로 배정 (60% 확률로 배양토)

[확장] 흙 종류 추가 (마사토, 바크 등)
[확장] 층별 배합 정책 (상단/중단/하단 다르게)
[확장] Worley Noise로 자연스러운 덩어리 패턴 배합
```

MVP에서 흙을 2종으로 제한하는 이유: 흙 종류가 많아지고 층별 정책까지 들어가면 최적화해야 할 변수가 폭발한다. 2종 단순 비율이면 "배양토가 많을 때 vs 펄라이트가 많을 때" 같은 명확한 비교가 가능하다.

## 3. [MVP] 에어룸 표현

### 3.1 형태

```
[MVP] 에어룸 = 정사면체 (2D 단면에서는 원형으로 근사)
  우리 핵심 IP가 정사면체이므로 형태는 정사면체로 고정한다.
  2D r-z 단면에서는 원형 영역으로 표현한다.

[확장] 다른 형태 비교 (캡슐형, 깔때기형 등)
[확장] 흙 유입 방지 구조 (루버, 메쉬 등)
```

형태를 정사면체 하나로 고정하는 이유: 우리 제품은 정사면체 에어룸이 거의 확정이다. 여러 형태를 비교하는 건 한참 나중 일이고, 지금 넣으면 변수만 늘어난다.

### 3.2 데이터 구조

```json
{
  "r": 4.0,
  "z": 8.0,
  "radius": 1.2,
  "pruning_zone_radius": 1.5
}
```

|변수|의미|단계|
|---|---|---|
|`r`, `z`|에어룸 중심 좌표 (cm)|[MVP]|
|`radius`|에어룸 단면 반경 (cm)|[MVP]|
|`pruning_zone_radius`|뿌리가 "닿았다"고 판정하는 영향권 반경 (cm). 보통 radius보다 약간 큼|[MVP]|

`pruning_zone_radius`를 따로 두는 이유: 뿌리 끝이 에어룸에 정확히 닿지 않아도, 충분히 가까우면 공기 노출로 프루닝이 일어난다. 이 "여유 반경"을 두면 더 현실적이다. 기본값은 `radius × 1.25` 정도.

### 3.3 에어룸 개수 제한 (중요)

화분 크기별로 현실적인 개수 상한을 둔다. 무한정 많이 넣으면 흙이 사라져서 식물이 못 자란다.

```
[MVP] 화분 크기별 에어룸 최대 개수:
  소형 (반지름 ≤ 8cm):  최대 6개
  중형 (반지름 ≤ 12cm): 최대 10개
  대형 (반지름 > 12cm):  최대 20개
```

이 제한은 랜덤 생성/최적화 시 반드시 지켜야 한다.

## 4. [MVP] Phase B — 뿌리 성장 + 에어프루닝

### 4.1 뿌리 모델

```
[MVP] 시작점: 화분 상단 중앙 (r=0, z=pot_height)
[MVP] 초기 중심뿌리: 1개 (메인 뿌리는 하나라는 가정)

세대 구분:
  1세대 (중심뿌리): 굵게, 주로 아래로 성장
  2세대 (곁뿌리):   중심뿌리에서 분기, 측면으로 탐색
  3세대 (실뿌리):   곁뿌리 끝 또는 프루닝 지점에서 폭발적 분기
```

`initial_roots` 기본값은 1. 재현성/안정성을 위해 3으로 둘 수도 있다(config에서 조정).

### 4.2 성장 로직

매 스텝마다 각 살아있는 뿌리 끝(팁)에 대해:

```
1. 방향 계산:
   [MVP] 방향 = 중력(아래) + 랜덤 노이즈
   [확장] + 수분 기울기 + 양분 기울기 (자원 확산 단계에서 추가)

2. 한 스텝 전진 (voxel_size 만큼)

3. 화분 밖이거나 바닥에 닿으면 → 팁 종료

4. 에어룸 접촉 검사 (4.3 참조)

5. 지나간 복셀의 root_visits += 1
```

세대별 성장 특성:

- 1세대: 중력 강하게(아래로), 노이즈 약하게(곧게)
- 2세대: 중력 약하게, 노이즈 강하게(옆으로 퍼짐)
- 3세대: 짧고 조밀하게 분기

### 4.3 에어프루닝 조건 (확률 기반)

병주의 가정: "에어룸을 만나면 프루닝이 거의 된다. 공기 조건은 나중에 SimScale이 구조적으로 잘 해줄 것이다."

따라서 MVP는 복잡한 습도/공기 조건 검사 없이 **단순 확률**로 처리한다.

```
[MVP] 뿌리 팁이 에어룸의 pruning_zone에 진입하면:
        프루닝 확률 = 0.85 (기본값, config 조정 가능)
        확률 통과 시 → 프루닝 발생

[확장] SimScale 환경 데이터로 확률 보정:
        P(pruning) = 0.85 × air_factor × dryness_factor
```

deterministic("만나면 무조건")이 아니라 확률로 두는 이유: 실제로는 모든 접촉이 프루닝으로 이어지지 않는다. 확률로 두면 나중에 SimScale 데이터(공기 흐름, 습도)로 이 확률을 정교하게 보정할 수 있는 여지가 생긴다.

### 4.4 프루닝 발생 시 동작

```
[MVP] 프루닝 발생:
  1. 현재 뿌리 팁 종료 (더 이상 직진 안 함)
  2. 종료 지점 뒤쪽 노드에서 다음 세대 뿌리 2~6개 분기
  3. 단, 최대 3세대까지만 (3세대는 더 분기 안 함)
  4. 프루닝 발생 위치 기록 (점수/시각화용)
```

세대 흐름: 1세대가 프루닝되면 2세대 곁뿌리들이 나오고, 2세대가 프루닝되면 3세대 실뿌리가 폭발한다. 한 번에 1세대 → 3세대로 건너뛰지 않는다(자연스러운 분기 순서).

### 4.5 뿌리 표면적 계산

```
[MVP] 각 뿌리 세그먼트의 표면적 합산:
  세그먼트 표면적 = 2π × 반경 × 길이
  세대별 반경: 1세대 굵게, 2세대 중간, 3세대 가늘게
  전체 표면적 = 모든 세그먼트 합

이 값이 클수록 좋다 (흡수 능력 ∝ 표면적).
```

## 5. [MVP] Phase C — 점수 계산 (G-Health Score)

### 5.1 MVP 점수 공식

검증할 항목을 최소화하기 위해 4개 항목만 쓴다. 각 항목의 가중치는 임의값이므로, 운영하며 조정한다(가중치 근거가 약하다는 걸 인지하고 시작).

```
[MVP] G-Health Score =
    + 뿌리 표면적 점수    (표면적이 클수록 좋음)
    + 프루닝 횟수 점수    (프루닝이 많을수록 잔뿌리 많음)
    - 흙 손실 패널티      (에어룸이 흙을 너무 많이 차지하면 감점)

  구체적으로:
    score = (root_surface_area × w1)
          + (pruning_count × w2)
          - (airroom_volume_ratio × w3)

  기본 가중치 (조정 가능):
    w1 = 1.0   (표면적, mm² 단위라 자연스럽게 큰 값)
    w2 = 5.0   (프루닝 1회당 가산점)
    w3 = 50.0  (흙 손실 비율 0~1에 곱하는 패널티)
```

### 5.2 점수에 넣지 않고 "결과 지표"로만 보는 것

GPT는 하단 프루닝 우선(lower > middle > upper)을 점수에 넣자고 했지만, MVP에서는 점수에 넣지 않는다. 가중치 근거가 또 필요해지기 때문이다. 대신 결과로 출력만 해서, 실제로 하단 프루닝이 중요한지 데이터로 먼저 확인한다.

```
[MVP] 결과 지표로만 출력 (점수 미반영):
  - 상/중/하 프루닝 횟수 분포
  - 흙 손실률
  - 총 에어룸 개수
  - 미사용 에어룸 수 (뿌리가 한 번도 안 닿은 에어룸)

[확장] 검증 후 필요하면 점수에 반영:
  - 하단 프루닝 우선 점수
  - 에어룸별 기여도
  - 수분 스트레스 / 산소 부족 (자원 확산 후)
```

### 5.3 점수 함수 구조 (확장 가능하게)

점수 항목을 나중에 추가하기 쉽도록 딕셔너리로 관리한다.

```python
score_components = {
    "surface_area": root_surface_area * w1,
    "pruning": pruning_count * w2,
    "soil_loss": -airroom_volume_ratio * w3,
    # 확장 시 여기에 항목 추가만 하면 됨
    # "lower_pruning_bonus": ...,
    # "water_uptake": ...,
}
total_score = sum(score_components.values())
```

이렇게 하면 항목을 추가/제거할 때 공식 전체를 안 건드려도 된다.

## 9. [확장] Phase G — 자원 흡수 (자리만)

MVP 완성 + 자원 확산 추가 후. 뿌리가 자원을 흡수하는 정량 계산.

```
[확장] MVP 흡수는 선형 모델로 시작:
  water_uptake = root_surface_area × local_theta × plant_water_factor
  oxygen_access = root_surface_area × local_oxygen

[확장] 그 다음 Michaelis-Menten으로 정교화:
  U = S × K × C / (Km + C) × Δt
```

처음부터 Michaelis-Menten을 쓰지 않는 이유: K, Km을 식물별로 확보하기 어렵다. 선형 모델로 시작해서 데이터가 쌓이면 정교한 식으로 교체한다.

### 9.1 [확장] 2D → 부피 환산 주의

2D r-z 단면으로 계산한 결과를 실제 흡수량(ml)으로 바꾸려면, 회전체 부피로 환산해야 한다(파푸스 정리). 중심축에서 먼 복셀일수록 실제로는 더 큰 부피를 차지하기 때문이다.

```
[확장] 복셀의 실제 부피 가중치 = 2π × r × (복셀 면적)
  → r축 거리에 비례한 가중치를 곱해야 정확
  → 이걸 빼먹으면 흡수량 수치가 통째로 틀림
```

이건 흡수량을 도입하는 순간 반드시 함께 적용해야 한다. MVP는 흡수량을 안 쓰므로 해당 없음.

# 내부 에어룸 배치 최적화를 위한 3차원 식물 뿌리 성장 및 생리적 분기 모델링 알고리즘 설계 보고서

식물 뿌리 시스템의 발달은 토양 내부의 수분, 산소, 양분과 같은 자원 분포뿐만 아니라 화분 내부의 물리적 경계 구조에 의해 결정된다. 특히 에어프루닝(Air-pruning, 공기 단근)은 뿌리 끝이 공기에 노출되었을 때 자연적으로 사멸하고, 그 결과 측면에서 잔뿌리가 폭발적으로 분기하는 생리적 현상이다. 가든위드카우치(Garden with Couch)가 개발 중인 내부 에어룸(Air Room) 배치 최적화 화분은 흙 속에 정사면체 형태의 빈 공간을 무작위 혹은 규칙적으로 배치하여 화분 전체 부피에 걸쳐 균일한 에어프루닝을 유도하는 혁신적인 지식재산권(IP)을 기반으로 한다.

그러나 이 시스템을 시뮬레이션하는 과정에서 단일 뿌리 노드로부터 비자연적이고 제어되지 않는 수준의 잔뿌리가 무한히 뻗어 나가는 '지하부 차원 폭발' 혹은 '폭발적 분기 버그'가 관찰된다. 이는 실제 식물이 가진 생리적 제약 조건과 호르몬 흐름, 위상적 공간 제약이 시뮬레이터 알고리즘에 정밀하게 반영되지 않았기 때문이다. 본 보고서는 이러한 문제를 해결하기 위한 식물 생리학 기반의 뿌리 성장 제약 알고리즘을 제안하고, 에어프루닝 팟 최적화를 위한 4단계 파이프라인 시뮬레이터의 상세 설계 및 수학적 정형화를 제공한다.

## 에어프루닝의 생물학적 메커니즘과 물리적 트리거 조건

시뮬레이터 내부에서 에어프루닝 현상을 물리적으로 타당하게 구현하기 위해서는 수분 포텐셜과 호르몬 흐름에 대한 정량적 임계치가 필요하다.

### 1. 물리적 탈수 및 세포 사멸 조건

뿌리 끝(Apical Meristem, 성장점)이 에어룸 내부의 공기층과 접촉하면, 국소적 상대습도($RH$)와 토양 매트릭스 포텐셜의 급격한 변화로 인해 탈수가 시작된다. 실험 데이터에 따르면 침엽수 및 활엽수 이식묘의 뿌리 수분 함량(Root Moisture Content, $RMC$)이 임계치인 180% 이하로 저하되거나, 뿌리 세포의 수분 포텐셜($\Psi$)이 $-0.5\text{ MPa}$ 이하로 떨어질 때 영구적인 세포막 손상과 조직 괴사가 발생한다.

$$\text{Pruning Trigger: } RMC < RMC_{crit} \quad (\approx 180\%) \quad \text{or} \quad \Psi_{root} \le \Psi_{crit} \quad (\approx -0.5\text{ MPa})$$

시뮬레이터 내부에서는 에어룸 경계면의 수분 국소 보너스 및 상대습도 데이터가 이러한 물리적 사멸 여부를 판정하는 직접적인 변수로 작용한다.

### 2. 식물 호르몬의 역류 및 잔뿌리 발달 기전

정상적으로 신장하는 뿌리에서는 줄기에서 합성된 옥신(Auxin)이 뿌리 중심부를 향해 하향 수송(Basipetal Transport)되어 뿌리 끝에 극대점(Auxin Maximum)을 형성한다. 이 고농도의 옥신은 정단우성(Apical Dominance)을 유지하며 성장점 부근에서의 측근(Lateral Root) 형성을 억제한다. 이와 동시에 성장점 세포에서는 측근 형성을 방해하는 호르몬인 사이토카닌(Cytokinin)이 지속적으로 합성되어 주변 성숙대로 방출된다.

그러나 에어룸 접촉으로 인해 뿌리 끝 세포가 사멸하면 다음과 같은 호르몬 변화가 발생한다:

- **정단우성의 상실 및 옥신 역류:** 성장점이라는 강력한 옥신 소비처(Sink)가 사라짐에 따라, 중심주를 통해 내려오던 옥신이 미사멸된 상부 성숙대로 역류(Retrograde flow) 및 축적된다.
    
- **사이토카닌 공급 중단:** 뿌리 끝의 사멸로 인해 국소적인 사이토카닌 농도가 급감하여 측근 형성 억제 작용이 해제된다.
    
- **측근 원기 활성화:** 상부 성숙대에 축적된 고농도의 옥신과 감소한 사이토카닌의 길항 작용에 의해, 내초(Pericycle) 세포가 자극받아 측근 원기(Lateral Root Primordia)의 세포 분열이 급격히 개시된다. 이 현상이 육안으로 관찰되는 '측면 잔뿌리의 폭발적 분기'이다.
    

## 뿌리 성장 제약 알고리즘: 무한 분기 현상 제어

단일 뿌리 노드에서 무제한으로 측근이 발생하는 컴퓨터 그래픽스 오류를 방지하기 위해, 학술용 뿌리 아키텍처 모델인 CRootBox 및 SimRoot의 위상적·공간적 제약 알고리즘을 도입해야 한다.

### 1. 위상적 구역화 알고리즘 (Topological Zoning)

뿌리 세그먼트는 물리적 위치와 연령에 따라 분기 가능 여부가 엄격히 차등화된다. 성장 중인 뿌리는 세 가지 영역으로 나뉜다 :

- **분열 구역(Apical Zone, $L_{apical}$):** 뿌리 끝 성장점으로부터 일정 거리 이내의 영역으로, 새로운 세그먼트가 추가되는 곳이다. 이 구역에서는 호르몬 억제 작용으로 인해 절대 측근이 발생하지 않는다.
    
- **분기 구역(Branching Zone, $L_{branch}$):** 분열 구역의 뒤쪽에 위치하며, 호르몬 균형 변화 시 측근 원기가 발달하여 외부로 뻗어 나올 수 있는 활성 구역이다.
    
- **기부 성숙 구역(Basal Zone, $L_{basal}$):** 줄기와 가장 가까운 오래된 구역으로, 이미 표피가 코르크화(Suberization)되어 추가적인 측근 형성이 불가능한 정적 영역이다.
    

따라서, 임의의 parent root가 생성할 수 있는 최대 측근 개수 $n_{ob}$는 다음과 같은 수식으로 제한된다 :

$$n_{ob} = \left\lfloor \frac{L_{parent} - (L_{apical} + L_{basal})}{l_n} \right\rfloor$$

여기서 $L_{parent}$는 parent root의 총 길이이며, $l_n$은 인접한 측근 사이의 최소 물리적 거리인 분기 간격(Inter-branching distance)이다. 이 수식을 적용하면 parent root의 전체 길이에 비례하여 분기 가능한 슬롯이 사전에 제한되므로 단일 노드에서의 폭발적 분기를 일차적으로 방단할 수 있다.

### 2. 물리적 분기 최소 간격 제약 (Inter-branching Distance)

시뮬레이터가 새로운 세그먼트를 추가할 때, 각 노드 주변에 구형의 '분기 금지 영역'을 설정한다. 임의의 노드 $\mathbf{X}_i$에서 이미 측근이 분기되었다면, 해당 노드로부터 거리 $l_n$ 이내에 있는 어떠한 인접 노드 $\mathbf{X}_j$에서도 새로운 분기가 시작될 수 없도록 강제한다.

$$\|\mathbf{X}_j - \mathbf{X}_i\| \ge l_n \quad \forall j \neq i$$

이 거리 조건은 한 점에 수십 개의 세그먼트가 엉키는 현상을 기하학적으로 원천 차단한다.

### 3. 분기 발달 지연 시간 알고리즘 (Tip Delay Time)

옥신 역류가 일어난 즉시 가시적인 잔뿌리가 돋아나는 것이 아니다. 세포 분열과 조직을 뚫고 나오는 물리적 신장 과정에는 시간적 지연이 수반된다. 이를 위해 시뮬레이터 내부에 '분기 지연 시간($t_{delay}$)' 변수를 도입한다.

에어룸 접촉에 의해 에어프루닝이 감지된 시간 $t_{prune}$ 이후, 실제 측근 세그먼트의 성장이 활성화되는 시간 $t_{active}$는 다음과 같이 확률적으로 결정된다 :

$$t_{active} = t_{prune} + t_{delay}$$

$$t_{delay} \sim \mathcal{N}(\mu_{delay}, \sigma_{delay}^2)$$

이 시간적 버퍼링을 통해 시뮬레이션의 프레임별 연산 부하가 분산되며, 시간 흐름에 따른 뿌리 구조의 자연스러운 점진적 팽창이 실현된다.

### 4. 국소 밀도 제약 (Local Density Suppression)

뿌리가 밀집된 환경에서는 한정된 양수분 자원으로 인해 성장이 스스로 제한된다. 시뮬레이터 공간을 복셀화(Voxelization)했을 때, 특정 복셀 내부의 뿌리 표면적 밀도 $\rho_{root}$가 임계 수준 $\rho_{max}$에 도달하면, 해당 복셀에 포함된 노드들의 분기 확률 $P_{branch}$를 강제로 감쇠시킨다 :

$$P_{branch} = P_{base} \cdot \max\left(0, 1 - \frac{\rho_{root}}{\rho_{max}}\right)$$

이 알고리즘은 화분 내부의 특정 구역이 기하학적으로 포화되는 현상을 효과적으로 방지한다.

## 뿌리 신장 역학 및 환경 주도 트로피즘(Tropism) 모델

뿌리 끝은 단순히 직선으로 자라지 않으며, 토양의 물리적 장벽과 자원 기울기에 반응하여 성장 방향을 실시간으로 수정한다.

### 1. 신장 속도 방정식

개별 뿌리 세그먼트의 시간에 따른 신장 속도 $dL/dt$는 식물의 내재적 노화 감쇠와 국소 토양 내 수분($C_{water}$) 및 산소($C_{oxygen}$) 농도에 의한 Michaelis-Menten식 제약을 따른다 :

$$\frac{dL}{dt} = r_{init} \cdot e^{-\beta \cdot t_{age}} \cdot \left( \frac{C_{water}}{K_{water} + C_{water}} \right) \cdot \left( \frac{C_{oxygen}}{K_{oxygen} + C_{oxygen}} \right)$$

여기서 $r_{init}$는 초기 신장 속도, $\beta$는 연령에 따른 속도 감쇠 계수, $t_{age}$는 해당 뿌리 축의 경과 시간이다. $K_{water}$와 $K_{oxygen}$은 각 자원에 대한 반포화 상수이다.

### 2. 굴성(Tropism)을 반영한 성장 방향 벡터 계산

새로운 노드가 생성될 방향을 나타내는 표적 방향 벡터 $\mathbf{v}_{target}$는 이전 세그먼트의 방향 유지성(강성), 중력굴성(Gravitropism), 수분굴성(Hydrotropism), 화학굴성(Chemotropism) 및 토양 불균일성에 따른 무작위 요소를 결합한 하중 벡터합으로 정의된다 :

$$\mathbf{v}_{target} = w_{dir} \mathbf{v}_{prev} + w_{grav} \mathbf{v}_{grav} + w_{hydro} \mathbf{v}_{hydro} + w_{chem} \mathbf{v}_{chem} + \mathbf{v}_{rand}$$

- $\mathbf{v}_{prev}$ : 직전 세그먼트의 정규화된 방향 벡터로, 생물학적 강성에 의한 직진성을 모델링한다.
    
- $\mathbf{v}_{grav} = [0, 0, -1]^T$ : 지구 중심을 향하는 중력 방향 벡터이다.
    
- $\mathbf{v}_{hydro} = \frac{\nabla C_{water}}{\|\nabla C_{water}\|}$ : 국소 복셀 간 수분 농도 기울기의 가장 가파른 방향을 향하는 수분굴성 벡터이다.
    
- $\mathbf{v}_{chem} = \frac{\nabla C_{nutrient}}{\|\nabla C_{nutrient}\|}$ : 양분 농도 기울기에 반응하는 화학굴성 벡터이다.
    
- $\mathbf{v}_{rand} \sim \mathcal{N}(0, \sigma^2 \mathbf{I})$ : 미세 토양 입자 충돌 및 내부 물리적 요동을 나타내는 무작위 가우시안 노이즈 벡터이다.
    
- $w_{dir}, w_{grav}, w_{hydro}, w_{chem}$ : 각 가중치는 식물 종류에 따라 다르게 부여된다.
    

CRootBox 프레임워크와 유사하게 시뮬레이터는 $\mathbf{v}_{target}$ 주위의 가우시안 원뿔 영역 $\text{Cone}(\mathbf{v}_{target}, \sigma)$에서 $N$개의 후보 벡터를 샘플링하고, 각 후보 중 물리적 제약을 위반하지 않으면서 굴성 적합도를 최대로 만족하는 경로를 실제 신장 방향으로 채택하는 스토캐스틱 최적화 기법을 취한다.

### 3. 경계면 장애물 충돌 해석 및 복구 메커니즘

화분 외벽이나 에어룸의 물리적 벽면에 충돌하는 현상은 부호화 거리 함수(Signed Distance Function, $SDF$)를 사용하여 판정한다 :

$$\Phi(\mathbf{X}) = \text{최인접 물리 경계 표면으로부터 노드 } \mathbf{X} \text{까지의 최단 거리}$$

$\Phi(\mathbf{X}_{next}) \le 0$이 되는 충돌 상황이 발생하면, 뿌리는 벽면을 관통할 수 없으므로 벽 표면의 법선 벡터 $\mathbf{n}$에 수직인 접평면 상으로 방향을 정사영하는 미끄러짐(Sliding) 알고리즘을 수행한다 :

$$\mathbf{v}_{resolved} = \mathbf{v}_{target} - (\mathbf{v}_{target} \cdot \mathbf{n}) \mathbf{n}$$

뿌리는 장애물 벽을 타고 미끄러지며 자라다가 장애물 영역을 벗어난 이후에도 즉시 원래의 굴성 벡터로 복귀하지 못한다. 벽면 접촉 시간 $t_{contact}$에 비례하는 지연 시간 $t_{recovery}$ 동안 촉수자극에 의한 일시적 성장 방향 왜곡을 유지하도록 모델링한다 :

$$t_{recovery} = \gamma \cdot t_{contact}$$

여기서 $\gamma$는 장애물 접촉 시 가해진 기계적 자극에 대한 식물 고유의 민감도 계수이다.

## GwC 에어프루닝 팟 시뮬레이터 v5 시스템 아키텍처

Garden with Couch의 에어룸 최적화 루틴은 4단계 파이프라인(Phase 1 ~ Phase 4)을 거치며 최적의 화분 구조를 유전 알고리즘으로 도출한다.

### Phase 1: 공간 복셀화 (Spatial Voxelization)

화분 내부 공간의 해상도를 0.5mm 수준의 3차원 그리드로 전부 분할하면 단일 화분당 약 2,700만 개의 셀이 생성되어 연산이 불가능해진다. 이를 해결하기 위해 두 가지 하이브리드 공간 제어 기법을 적용한다 :

1. **다단계 격자 정밀화(Multi-scale Zoom-in):** 유전 알고리즘의 초기 거대 탐색 단계에서는 1.0cm 단위의 성긴 복셀 격자를 사용해 전체 탐색을 수행하고, 적합도가 높은 상위 5개 후보안에 대해서만 0.5mm의 고정밀 격자로 공간을 재분할하여 정밀 시뮬레이션을 수행한다.
    
2. **2D r-z 회전 대칭 근사:** 에어룸 구조물이 방사 대칭형으로 배치된 설계안의 경우, 3D 풀-그리드 계산 대신 원통 좌표계의 $r\text{-}z$ 단면(2차원 단면)만을 시뮬레이션함으로써 연산 차원을 극적으로 축축한다. 이는 연산량을 360분의 1로 감소시켜 실시간성에 가까운 계산 속도를 제공한다.
    

### Phase 2: Cellular Automata (CA) 자원 확산

화분 상부에서 물 500ml를 주입하는 시점을 기점으로, 중력 가속도와 기질별 통기성에 따른 수분, 양분, 산소의 확산 방정식을 계산한다. 3차원 유체 거동을 완전한 CFD(Computational Fluid Dynamics)로 1000회 이상 반복 연산하는 것은 무거운 전산 리소스가 소요되므로, 1000배 이상 빠른 격자 기반 '셀룰러 오토마타(Cellular Automata)' 확산 공식을 활용한다. 에어룸 공간에 해당하는 복셀들은 수분 확산의 장벽(우회로)으로 작동하게 구현하여 공기층 주변의 수분 증발 상태를 정적 모델링한다.

### Phase 3: 뿌리 흡수 모델링 (Mass Balance)

자원의 실제 흡수량 $U$는 단순 표면적 합산이 아닌, 뿌리가 차지하는 격자 내 물리적 자원 농도 $C$와의 상호작용적 물질수지(Mass Balance)에 의해 계산된다 :

$$U(x, y, z, t) = S_{root} \times K_{sector} \times C(x, y, z, t) \times \Delta t$$

뿌리 끝이 에어룸을 만나 에어프루닝이 일어날 때마다 잔뿌리가 폭발적으로 신장하여 국소 표면적 $S_{root}$가 가파르게 상승한다. 이 상승된 표면적이 자원 농도 밀도와 만물릴 때 물리적인 흡수 성과(ml, mg 단위)로 정량화된다.

### Phase 4: 유전 알고리즘(GA) 최적화

DEAP(Distributed Evolutionary Algorithms in Python) 라이브러리를 기반으로 최적의 에어룸 3차원 배치 좌표를 탐색한다. 1000개의 무작위 화분 구조군(Population)을 생성하고 세대별 교차(Crossover) 및 돌연변이(Mutation)를 수행한다. 유전 알고리즘이 특정 지역 최적해(Local Minima)에 고착되는 것을 방지하기 위해 다음과 같은 수렴 보장 수단을 병행한다:

- 성장이 정체될 때마다 돌연변이율을 동적으로 자동 상향한다.
    
- 5회 이상의 완전 독립 실행(Independent runs) 결과를 종합 평균하여 편향을 상쇄한다.
    
- 적합도 평탄 영역 도달 시 얼리 스토핑(Early Stopping)을 발동하여 연산 자원을 보호한다.
    

## 시뮬레이션 매개변수 및 보정 데이터 세트

신뢰도 높은 시뮬레이션을 구현하려면 토양 물리 속성, 식물 생리학 파라미터, 성능 채점 기준이 고도로 튜닝되어야 한다.

### 1. 흙 배합 기질별 물리 특성 데이터

각 격자 복셀에 입력될 흙 종류별 기초 물성 매트릭스이다.

|**기질 종류**|**보수력 (Moisture Capacity)**|**통기성 (Air Porosity)**|**시뮬레이션 내 물리적 거동 성향**|
|---|---|---|---|
|**배양토**|$0.9$|$0.1$|높은 보수력을 유지하며 자원의 완만한 확산을 유도|
|**펄라이트**|$0.3$|$0.8$|중력 배수를 촉진하고 국소 산소 도달율을 극대화|
|**마사토**|$0.4$|$0.6$|거친 입자로 물리적 슬라이딩 방향 전환 유도|
|**바크**|$0.2$|$0.8$|통기성이 우수하며 수분 증발 속도를 높임|
|**에어룸 (공기층)**|$0.0$|$1.0$|수분 통과를 완전히 차단하며 접촉 시 에어프루닝 작동|

## 하드웨어 공학적 설계 가이드라인

시뮬레이션에서 수렴된 3차원 에어룸 좌표와 사양을 실제 플라스틱 사출 금형 및 3D 프린터용 설계 도면(Fusion 360)으로 변환하기 위한 하드웨어 공학 규칙이다.

### 1. 에어룸 기하 구조 및 크기 제약

에어룸 한 변의 길이는 실물 제조 공차 및 뿌리 사멸 감도 특성을 극대화하기 위해 $0.5\text{ cm} \sim 2.0\text{ cm}$ 범위 내로 설계 제약조건을 설정해야 한다. 실생활에서 사용 시 엄지손가락 한 마디 크기 수준인 $2.0\text{ cm} \sim 2.5\text{ cm}$ 규격을 초과할 경우 화분 내부 토양 부피가 심각하게 손실되어 수분 보유력이 상실된다. 화분 스케일별 탑재 가능한 물리적인 에어룸 개수의 안전 한계선은 다음과 같이 권장된다:

- 소형 화분 (S, 지름 10~15cm): 최대 $6\text{개}$
    
- 중형 화분 (M, 지름 15~20cm): 최대 $10\text{개}$
    
- 대형 화분 (L, 지름 20cm 이상): 최대 $20\text{개}$
    

그 이상의 과밀 배치는 흙의 연속적인 흐름을 방해하여 뿌리가 흙 전체로 영양을 흡수하기 어렵게 만든다. 에어룸 외형은 기본 정사면체(Regular Tetrahedron) 구조를 채택하되, 적층 설계 시 깔때기형(Truncated Cone), 벌집형(Honeycomb), 원통 캡슐형, 사각뿔형 등을 선택할 수 있게 하여 경계에서의 충돌 접촉각을 튜닝한다.

### 2. 충돌 반경 스케일링 공식

구조물 표면 근처에서 물리적으로 닿지 않았으나 통기성 전단 구역에 진입하는 순간의 에어프루닝을 모사하기 위해, 기하학적 형상의 충돌 한계 반경($R_{collision}$)을 계산해야 한다.

$$R_{collision} = L_{side} \times C_{factor}$$

여기서 $L_{side}$는 에어룸 구조물의 한 변의 길이이며, $C_{factor}$는 충돌 인자로 $0.25 \sim 0.40$ 사이의 값을 사용하되 기본값으로는 균형형 수치인 $0.35$가 권장된다. 이 값이 과도하게 낮으면 뿌리가 구조물을 뚫고 지나가는 관통 현상이 시뮬레이션상에 연출되며, 과도하게 높으면 공기 방울 근처에도 가지 않은 뿌리가 프루닝되어 탈수되는 논리적 모순이 발생한다.

### 3. 토양 유입 원천 차단 설계 (Soil-shielded Louver AirRoom)

에어룸 내부에 흙이 들어와 메워지면 통기 구조가 상실되므로 , 실제 부품 설계 시 다음 설계 규칙을 의무 적용한다:

- **상부 폐쇄 설계:** 낙하하는 흙 압력을 수직 방향에서 온전히 견디도록 상부를 밀폐한다.
    
- **측면 루버 슬릿 (Louver Slit):** 공기 교환은 가능하되 흙 알갱이는 빗겨 나가도록 하향 각도의 격자 루버 슬릿을 설계한다.
    
- **하부 미세 배수 슬롯:** 슬릿 사이로 우연히 유입된 수분이나 미세 먼지가 고이지 않고 아래로 배출되도록 바닥면에 미세 배수공을 천공한다.
    

## 알려진 시스템 문제점 및 해결 방안 분석

시뮬레이터 고도화 과정에서 직면하게 될 대표적인 한계점과 대응 설계 방법론이다.

### 1. 자원 확산 방정식 단순화에 따른 정밀도 괴리

셀룰러 오토마타(CA) 기반 확산은 연산이 빠르지만 고속의 수분 흐름이나 압력 불균형을 온전히 추종하지 못한다.

- _해결 방안:_ 시뮬레이터 내에서 유전 알고리즘으로 최종 선별된 최우수 설계안 Top 5 후보들에 대해서만 전용 CAD 파일로 추출하여 SimScale 등의 풀-CFD 상용 툴을 활용한 이중 물리 검증 체계를 구현한다.
    

### 2. 유전 알고리즘의 지역 최적해 고착 문제

화분 기하 내부의 에어룸 최적 배치 문제는 전역 적합도 곡면이 불연속적이고 거칠기 때문에 지역 해에 갇히기 매우 쉽다.

- _해결 방안:_ DEAP 내에 아일랜드 모델(Island Model) 분리 탐색 구조를 도입하여 독립적인 유전자 풀을 동시 진화시키며 주기적으로 우수 유전자를 이주시키는 아키텍처를 도입한다.
    

### 3. 2D r-z 회전 대칭의 비대칭형 구조 표현 한계

2D 단면 단순화는 완벽한 방사 대칭형 에어룸 배치에서만 기능하며, 나선형 배치나 분산형 정사면체 배치 등 비대칭형 독창적 형상 구조를 평가하지 못한다.

- _해결 방안:_ 1단계 탐색 시 거친 3D 복셀 해상도($1.0\text{ cm}$) 탐색과 2D 회전 대칭 탐색을 선택하여 가동할 수 있도록 실행 스위치를 유연화된 클래스로 구현한다.

## 결론 및 실무 제언

본 보고서에서 제시한 3차원 식물 뿌리 성장 및 에어프루닝 제약 알고리즘은 'Garden with Couch' 에어프루닝 팟의 기하학적 설계 한계를 정밀하게 축소하기 위한 강력한 의사결정 지원 도구이다. 시뮬레이터 개발자는 100% 생물학적 뿌리를 완벽히 똑같이 그리는 것을 목표로 삼기보다는, 화분 내부의 에어룸 최적 배치 후보군을 빠르게 좁히는 정량적 비교 수단으로 활용해야 한다.

특히 시뮬레이터 구현 시 다음 원칙을 반드시 고수해야 한다:

1. **에어프루닝 발생 0회 설계안은 강력한 벌점 부여를 통해 즉시 도태시킨다.** 표면적 값($surface\_score$)만 크게 계산된 '에어프루닝 우회 설계안'이 높은 점수를 차지하는 컴퓨터 오류를 방지하기 위함이다.
    
2. **화분 내 물리적 에어룸 개수는 최상의 양산성과 토양 부피 효율을 보장하는 한계치인 10개(중형 기준) 이하로 유지한다.**
    
3. 성장 제약 조건인 '위상적 구역화'와 '분기 간격 최소화'를 수학적으로 필수 반영한다. 이를 생략하면 시뮬레이션 가동 즉시 뿌리가 일그러지며 무한 전산 루프에 걸려 시스템 전체가 다운될 것이다.
    

이러한 수치적 가이드라인과 생물학적 제약 조건을 체계적으로 적용한 시뮬레이터를 기반으로, 정밀하고 검증된 내부 에어프루닝 기능성 화분의 성공적인 시장 론칭과 IP 장벽 구축이 가능할 것으로 판단된다.

## 부록 A: 현재 MVP 시뮬레이터 분석 및 개선 방향 (2026-05-23 M3 단계)

### A1. 2D r-z 단면도 읽는 법

현재 시각화는 화분을 위에서 내려다본 게 아니라 **중심축으로 세로로 반 자른 단면**이다.

```
화분 실물 (원통)         2D r-z 단면도
┌──────┐                z↑
│      │  20cm ──────────┤상단
│      │                │
│  ╭──╮│  15cm          │  ⭕ ← 에어룸 (2D에선 원)
│  │  ││                │
│  ╰──╯│  10cm          │  ~~~ ← 뿌리 경로
│      │                │
│      │   5cm          │  ● ← 프루닝 지점
│      │                │
└──────┘   0  ──────────┤바닥
  ←12cm→       r→ 0     6     12cm
  (지름 24cm)      (중앙)      (벽)
```

- **x축(r)**: 화분 중심에서 벽까지 거리 (0~12cm). 오른쪽으로 갈수록 벽에 가까움
- **y축(z)**: 바닥에서 높이 (0~20cm). 위로 갈수록 화분 상단
- **회전 대칭 가정**: 이 단면을 z축 중심으로 360° 돌리면 3D 화분이 됨
  - r=2cm에 있는 점 → 실제로는 반지름 2cm인 원(고리)
  - r=0에 있는 점 → 화분 정중앙
- 갈색 = 배양토, 베이지 = 펄라이트, 흰 원 = 에어룸, 초록선 = 뿌리, 빨간점 = 프루닝

### A2. 프루닝이 lower에만 집중된 원인 분석

M3 테스트 결과: 34회 프루닝 전부 "lower zone" (z축 하단 1/3, 0~6.7cm)에서 발생.

**근본 원인: 1세대 뿌리가 거의 직진만 함**

| 항목 | 현재값 | 영향 |
|------|--------|------|
| 1세대 스텝당 노이즈 | ±3° | 50스텝 성장 후 r축 최대 편차 ≈ 1cm |
| 1세대 최대 각도 | ±25° | 있어도 노이즈가 워낙 작아 도달 못 함 |
| 분기 시 2세대 각도 | ±60° | 분기 자체는 넓게 퍼지나, 분기가 발생한 위치가 이미 lower zone |
| 에어룸 r 범위 | 1~11cm (랜덤) | 10개 중 8개는 뿌리가 아예 못 만남 (r=0 근처만 지나감) |

**결론**: 현재 알고리즘은 뿌리가 화분 전체 부피를 탐색하지 못하고 중심축만 타고 내려간다. 10개 에어룸 중 2개만 우연히 중심 근처에 있어서 그 2개에서만 연쇄 프루닝 발생 → lower zone에 몰림.

### A3. 뿌리 성장 알고리즘 개선 옵션 (MVP 즉시 적용 가능)

#### 옵션 1: 트로피즘 가중치 조정 (쉬움, 코드만 수정)
1세대 뿌리의 측면 확산을 늘려서 더 넓은 영역 탐색:

```
현재:       noise=±3°, max_angle=25°
개선:       noise=±8°, max_angle=40°
```

이렇게만 바꿔도 1세대 뿌리가 r=0~4cm 영역을 커버하게 됨.

#### 옵션 2: 초기 뿌리 개수 증가 (쉬움, config 수정)
`initial_roots: 1` → `initial_roots: 3`
3개 뿌리가 상단에서 조금씩 다른 위치에서 시작하면 각자 다른 경로로 성장.

#### 옵션 3: 2세대 분기 각도를 90°까지 확대 (쉬움)
현재 BRANCH_ANGLE_RANGE[2] = (-60°, 60°) → (-85°, 85°)
프루닝된 지점에서 옆으로 더 퍼지게 함.

#### 옵션 4: 옥신 분기 지연 + 분기 간격 제한 (중간, CRootBox 스타일)
CRootBox/SimRoot 스타일의 위상적 구역화 도입:
- 분기 간격 최소 거리 (inter-branching distance): 0.5cm
- 1개 parent root가 생성할 수 있는 최대 분기 수 제한
- 분기 지연 시간 (branch delay): 프루닝 후 바로 분기하지 않고 n스텝 대기

#### 옵션 5: 굴성(Tropism) 방정식 도입 (어려움, 확장 단계)
v_target = w_dir·v_prev + w_grav·v_grav + w_hydro·v_hydro + v_rand
현재 확장 슬롯인 diffusion/uptake를 활성화해야 필요해지는 옵션.

**권장**: M4(pipeline) → M5(search)까지는 **옵션 1+2**만 적용해서 진행하고, 점수 함수 검증 후 옵션 3~4를 적용한다.

### A4. 3D 시각화 옵션

현재 2D r-z 단면을 3D로 변환하려면 회전체(Solid of Revolution) 방식 사용:

```python
# 2D r-z 데이터 → 3D 회전체
theta = np.linspace(0, 2*np.pi, 36)  # 36각형
R, T = np.meshgrid(r_coords, theta)
X = R * np.cos(T)
Y = R * np.sin(T)
Z = np.tile(z_coords, (len(theta), 1))

# matplotlib 3D
fig = plt.figure()
ax = fig.add_subplot(111, projection='3d')
ax.plot_surface(X, Y, Z, ...)
```

| 방식 | 장점 | 단점 | 적용시점 |
|------|------|------|----------|
| **mplot3d 회전체** | matplotlib만 있으면 됨, 코드 간단 | 인터랙티브 약함, 많은 데이터 느림 | M4 이후 |
| **Plotly** | 드래그/줌 가능, HTML export | 추가 설치 필요 | M5 이후 (IR용) |
| **Mayavi** | 고퀄리티 볼륨 렌더링 | 설치 복잡, Windows 호환성 | 확장 |

**3D 시각화는 M4(pipeline) 완성 후 추가 예정.** 지금은 2D 단면도가 더 보기 편하고 디버깅에도 유리함.
